# v0.1.136 — Workers Menu & UI Cleanup

## Highlights
Worker management now has its own menu — a right-edge tab drawer, just like the Buildings menu. The old Inventory window has been removed.

## New
- **Workers drawer**: A new `🧑` tab on the right edge of the screen opens the worker list. Clicking a worker shows the detail view as a sub-menu — built like the Buildings menu (back bar, name with ✏️ rename, efficiency readout in the same style).
- **Per-tile consume**: The old "Feed now" button is gone. Instead, each tile (food *and* every perk) has its own **Consume** button — exactly as wide as the tile. It consumes the food or the perk good immediately, without waiting for the day change. The button is disabled when the supply is already active, under-assigned, or out of stock.
- **Level-up button**: The small ⬆ icon in the header is gone; instead there's a clear **Level up** button at the bottom of the worker detail view (appears whenever a level-up or an open perk choice is pending).

## Changed
- **Inventory menu removed**: The Inventory window opened with `I` / `Tab` (with its Inventory and Crafting tabs) has been removed entirely, along with the `I` and `Tab` keybinds. Crafting still works per building via the crafting-bench button in the building detail view.
- The worker list has moved out of the Inventory window into the new Workers drawer.

## Fixes
- **Picking a perk no longer closes the menu**: Previously, selecting a perk card closed the open drawer in the background. Drawers now stay open while a full-screen dialog (such as the perk picker) is active.

## Technical
- New edge drawer under `src/ui/edge_drawer/npcs/` (list + detail) plus `NpcsDrawer` (layer 24), mirrored from the Buildings drawer.
- Removed the old `NpcDetailPanel` (floating window) and `InventoryScreen`.
- New `NPCSystem.consume_perk_now()` call for immediately consuming perk goods.
- `EdgeDrawerController` suppresses its auto-close while a modal in the group `blocks_edge_drawer_autoclose` is visible.

## Known Issues
- Changes have not yet been verified headless through the test suite / Godot editor.
