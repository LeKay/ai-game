# AI Asset Generation Prompts — Ore & Gem Deposits

Prompts for PixelLab. Mirrors the clay deposit assets (`fertility-prompts.md`) for the six new
mineable deposits: **iron, copper, tin, silver, gold, gemstones**. Same mechanic as clay — hidden
at generation, exposed by Search, revealed as a pit tile, hand-mined.
Spec: `design/quick-specs/ore-gem-deposits-2026-06-22.md`. Base rules: `sprite-prompts.md`.

**Endpoint per asset type (same split as clay/wheat):**
- **Ground/terrain tiles** (edge-to-edge, tileable — the revealed pit): **`create-tiles-pro`**
  `POST https://api.pixellab.ai/v2/create-tiles-pro` (async — returns `background_job_id`; poll
  `GET /v2/background-jobs/{id}` until `status == "completed"`). Fields: `tile_type
  "square_topdown"`, `tile_size 64`, `tile_view_angle 90.0`, `tile_depth_ratio 0.0`, palette inline
  in `description`. Completed job exposes **16 tiles** at `last_response.images[i].base64` — these
  are **raw RGBA8** (`w*h*4` bytes), NOT PNG, so encode via Pillow before saving. Saved **all** as
  `env_tile_<ore>_NN.png` and wired into `terrain_renderer.gd` → `_TERRAIN_PNG_VARIANTS[<TileType>]`
  (already wired). Generated by `generate_ore_ground_tiles.py`.
- **Centered single assets** (overlay/world icons + UI icons): pixen endpoint `create-image-pixen`,
  transparent background. Generated by `generate_ore_deposit_assets.py`.

Each pit must be a top-down (90°) opaque ground tile filling the whole tile, edge-to-edge, with the
ore visibly embedded as veins/nuggets/crystals so the resource is readable from above and clearly
distinct from plain stone (gray) and clay (reddish-orange).

## Ground/terrain tiles (tileset endpoint, 16 variants each)

Common template — substitute the ore-specific line + palette:

```
Exposed <ORE> ore pit ground seen from directly straight above (90-degree top-down view),
covering the entire tile from edge to edge with no gaps and no transparency — the surface is the
rocky pit floor itself, opaque in every corner. The base is broken grey-brown excavated rock. <ORE_DETAIL>
The veins and nuggets are scattered across the interior, 6 to 10 small marks each 1 to 3 pixels,
distributed evenly, not clustered in one spot. Light comes from the upper-left: upper-left slightly
lighter, lower-right slightly darker. All marks stay at least 3 pixels from every tile edge so
nothing appears cut off when tiles are placed side by side. No grass, no plants, no tools, no objects.
```

| Ore | `<ORE_DETAIL>` | Palette (highlight · base · shadow · ore-accent) |
|-----|----------------|--------------------------------------------------|

## Overlay / UI icons (pixen endpoint, transparent)

Generated by `generate_ore_deposit_assets.py` (proven pixen pattern). Two files per ore:
- `assets/art/tiles/env_tile_resource_<ore>.png` — world/carry overlay (`world_icon_path`)
- `assets/ui/icons/resources/<ore>.png` — inventory/HUD icon (`icon_path`)

Each is a single centered ore chunk / nugget cluster (gemstones = a small cluster of cut crystals),
~60–70° elevated view, fully visible with ≥4 px clearance, light upper-left, transparent background.
Per-ore palettes match the ground-tile accents above. See the script for the exact strings.
